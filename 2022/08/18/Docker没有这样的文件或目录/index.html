如何解决Windows环境下在Docker中运行*.sh，提示没有文件的问题

最近在尝试弄一个私有云相册，需要一个能够满足自动标记和查找图片并且简单可靠的应用，最好呢？是可以全平台的，PhotoPrism 便成为了我最好的选择，部署完后我发现我可能遇到了点麻烦。

如果你使用Windows系统，并且在Docker中运行一些*.sh文件，你可能会遇到这样的错误：
Container command '/start.sh' not found or does not exist
或者
'bash\r': No such file or directory
为啥呢？怎么办呀？本文将详细分析问题所在，并给出一些简单的解决方案。

问题原因

这个问题的根本原因是文件格式。Windows系统和Linux系统使用不同的 换行符 来表示文件的结束。Windows系统使用回车换行（CR-LF），而Linux系统使用换行（LF）。如果你在Windows系统下编辑或保存了一个*.sh文件，它可能会自动转换为CR-LF格式，而这个格式对于Linux系统来说是无法识别的。

当你把这个*.sh文件放到Docker容器中运行时，Docker会尝试用/bin/sh来执行它，但是由于文件格式不正确，它会认为这个文件不存在或者不是一个可执行命令。

解决方案

解决方案很简单，就是把*.sh文件转换为LF格式。有很多方法可以做到这一点，比如：

  • 使用一些文本编辑器（如Sublime Text, VS Code等）来修改文件的换行符设- 置，并重新保存。
  • 使用一些命令行工具（如dos2unix, sed等）来转换文件。
  • 使用git配置（core.autocrlf=false）来避免自动转换换行符。

总结
